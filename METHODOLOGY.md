## 一、Harness Engineering 核心原则

### 1. 明确意图 (Intent)
**AI 不是猜心高手，是执行机器。**

- ✅ "创建一个响应式导航栏，包含 Logo、首页、课程、关于、联系我们 5 个链接"
- ❌ "做个好看的导航栏"

### 2. 约束输出 (Constraint)
**没有约束的 AI = 随机数生成器。**

- 限定技术栈：Vue 3 + Tailwind
- 限定代码风格：ESLint + Prettier
- 限定文件结构：src/components/、src/views/、src/assets/

### 3. 迭代验证 (Iterate)
**小步快跑，步步为营。**

- 每次只做一个功能点
- 完成 → 提交 → 部署 → 验证
- 发现问题立即修复

### 4. 分层控制 (Layer)
**从宏观到微观，不跳跃。**

```
Level 1: 整体架构（目录结构、技术选型）
Level 2: 页面布局（组件划分、路由设计）
Level 3: 组件实现（具体代码、样式、交互）
Level 4: 细节优化（动画、响应式、性能）
```

## 二、OpenClaw 开发工作流

### 角色分工

| 角色 | 负责 | 使用模型 |
|:---:|:---|:---|
| **架构师** | 整体设计、技术决策 | Qwen |
| **开发者** | 代码生成、实现 | Qwen / DeepSeek |
| **审查员** | 代码质量、安全 | Qwen |
| **运维** | 部署、监控 | Qwen |
| **文案** | 内容撰写、优化 | Kimi |

### 开发流程

```
[用户] 描述需求
   ↓
[OpenClaw] 分析需求 → 拆解任务
   ↓
[架构师] 输出架构方案 → 用户确认
   ↓
[开发者] 逐模块编码 → 提交 Git
   ↓
[审查员] 代码审查 → 通过/打回
   ↓
[运维] 部署到生产 → 验证
   ↓
[用户] 最终验收
```
