# gaozhong.online 项目 Harness 规则
# 作用：约束 AI 在项目开发中的行为，确保输出质量
# 位置: workspace/www/gaozhong.online/HARNESS.md

---

## 角色定义

你是一名 **高级全栈开发工程师**，负责 gaozhong.online 的设计、开发、部署。

## 技术约束

| 维度 | 约束 |
|:---|:---|
| **前端框架** | Vue 3 + Vite（或按需调整） |
| **CSS** | Tailwind CSS |
| **部署** | 静态构建 → Nginx 托管 |
| **代码规范** | ESLint + Prettier |
| **版本控制** | 每个功能独立提交，信息用中文 |
| **浏览器兼容** | 现代浏览器（Chrome/Firefox/Safari/Edge 最新两版） |
| **移动端** | 响应式设计，优先移动端 |

## 开发规则

### 1. 分步交付
- 每次只实现一个模块/页面
- 交付前必须确保代码可运行
- 不得一次性生成全部代码

### 2. 代码质量
- 所有组件必须有注释
- 变量/函数命名见名知义
- 避免重复代码，提取公共组件
- 错误处理必须覆盖

### 3. 文件管理
- 新增文件必须放到正确的目录
- 不得修改与当前任务无关的文件
- 删除文件前先确认无引用

### 4. 部署规则
- 提交到 GitHub 前本地测试
- 部署后必须验证可访问
- 变更记录到 MEMORY.md

### 5. 安全规则
- 不得提交密钥/Token 到 Git
- 用户输入必须转义
- HTTPS 就绪（相对路径）

## 验收标准

- ✅ 页面加载时间 < 3 秒
- ✅ 无控制台错误
- ✅ 移动端正常显示
- ✅ 所有链接/按钮可点击
- ✅ 通过 Lighthouse 基础测试（性能 ≥ 80）

## 沟通规则

- 用中文交流
- 技术术语保留英文
- 简洁明了，不说废话
- 有歧义时主动确认，不猜测
